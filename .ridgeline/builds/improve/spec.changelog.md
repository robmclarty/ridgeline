# Spec Changelog

## Iteration 1

Refine was attempted twice by `ridgeline refine improve` and both runs hit the harness context-window limit before writing output. This iteration was completed manually by incorporating all 16 Active Recommendations from `research.md` into `spec.md`, following the refiner prompt guidelines (additive, preserve structure, cite sources inline, do not modify constraints.md or taste.md, flag conflicts).

### Applied

- **Prompt caching — full rewrite of acceptance criteria** (research Rec 1, 4, 11, 12, 15). The Claude CLI has no `--cache-breakpoint` / `--cache-control` argv flag — `cache_control` is an HTTP-body directive that only the CLI itself emits. Replaced the argv-assertion criterion with: (a) write the stable block to a temp file passed via `--append-system-prompt-file`, (b) pass `--exclude-dynamic-system-prompt-sections` on `-p` invocations so per-machine sections (cwd/date/git status) don't poison the prefix, (c) detect flag availability via `claude --help` at startup and no-op if absent, (d) assert byte-identity on the temp file contents across runs (not an in-process slice), (e) delegate cache invalidation to the upstream server-side content-hash — no client-side `cache-key.json`, (f) log `cache_creation_input_tokens` and `cache_read_input_tokens` to `trajectory.jsonl` so caching is measurable, (g) preflight warns when the combined stable block is under the minimum cacheable prefix (4,096 tokens Opus-family / 2,048 Sonnet). Sources: platform.claude.com prompt-caching docs, code.claude.com CLI reference, Claude Code 2.1.98 changelog, claudecodecamp.com.
- **a11y and vision as Playwright consumers, not peers** (research Rec 2). axe-core requires a real DOM and documented behavior is that JSDOM disables the `color-contrast` rule — a primary reason to ship the sensor. Clarified that `a11y.ts` and `vision.ts` are consumers of the Playwright `Page`; when Playwright is unresolvable, they each emit a warning `SensorFinding` with the install hint rather than falling back to JSDOM. Contrast sensor stays independent (static hex scoring). Chose direct `page.addScriptTag({ path: require.resolve('axe-core') })` injection over the `@axe-core/playwright` wrapper because the wrapper tracks axe-core's `major.minor` non-SemVer-compatibly. Sources: dequelabs/axe-core-npm playwright README, axe-core issues #1167 / #1417, npm wcag-contrast, jest-axe.
- **Playwright install hint includes browser binaries** (research Rec 9). Updated the preflight install-hint acceptance criterion from `npm install --save-dev playwright` to `npm install --save-dev playwright && npx playwright install chromium` — the package install alone is one step short of a usable browser (~150 MB binary). The Playwright sensor also catches "browser not found" on first launch and emits the same hint. Source: playwright.dev/docs/intro.
- **Sandbox / Chromium launch interaction** (research Rec 10). When a sandbox environment is detected (Greywall on macOS, `BWRAP_DETECTED` on Linux), Playwright launches Chromium with `args: ['--no-sandbox', '--disable-setuid-sandbox']` and a 10 s launch timeout; launch failure emits a warning `SensorFinding` containing `sandbox-incompatible` and the phase continues rather than aborts. Source: chromium.googlesource.com Linux sandboxing docs.
- **shape.md `## Runtime` section convention** (research Rec 3). `shape.md` has no YAML front matter today (verified at `src/commands/shape.ts:14-172`), so added a new trailing `## Runtime` section with the literal bullet format `- **Dev server port:** 5173` and the parse regex `/^\s*-\s*\*\*Dev server port:\*\*\s+(\d+)\s*$/m`. Extends `SHAPE_OUTPUT_SCHEMA` with an optional `runtime?: { devServerPort?: number }` field. Explicitly rejected YAML front matter (ecosystem specialist's alternative) because it would break the project's pure-Markdown shape.md convention. Source: codebase inspection.
- **SSE named-event schema** (research Rec 5). Named the three event types (`event: state`, `event: budget`, `event: trajectory`), added monotonic `id:` for `Last-Event-ID` reconnect, `retry: 2000` on stream open, `: heartbeat\n\n` comment every ~20 s, explicit response headers (`Content-Type: text/event-stream`, `Cache-Control: no-cache, no-transform`, `Connection: keep-alive`, `X-Accel-Buffering: no`), and an in-memory replay buffer of ≥200 events per type for reconnect recovery. Sources: MDN SSE guide, WHATWG HTML SSE spec, speakeasy.com, tigerabrodi.blog.
- **Efficient state-file watching** (research Rec 14). For `state.json` / `budget.json`: `fs.watch` with 50 ms trailing-edge debounce + changed-value diff — no spurious dispatches. For `trajectory.jsonl`: `fs.watch` + byte-offset tracker, seek-and-read only appended lines, never re-read the whole file. Explicitly avoids `fs.watchFile` polling (5 s default is slower than the 2 s polling fallback). Source: nodejs.org fs API docs.
- **Favicon simplification** (research Rec 13). Replaced "debounced to once per actual status change" with "the favicon `href` is only reassigned when the mapped color actually changes (last-value compared in client JS)" — browsers no-op identical data-URI assignments automatically, so a time-based debounce is unnecessary. Source: MDN `<link>` docs, WHATWG HTML rel=icon.
- **Deterministic HSL brightening algorithm** (research Rec 4). `wcag-contrast` is measurement-only and ships no brightening helper — the "brightened if short" requirement was underspecified. Added `src/ui/contrast.ts` exporting `brightenForContrast(accentHex, bgHex, targetRatio=4.5)`: composite 10 %-opacity accent over background → HSL → step L upward in 2 % increments (cap 95–98 %) → remeasure until target met, fall back to `--text #E5E7EB` on loop cap. All adjustments computed at build time and baked into served CSS. Vitest confirms cyan already clears 4.5:1 on the specified palette (~7.5:1), so the loop is a fallback for future accent edits. Sources: github.com/tmcw/wcag-contrast, accessiblepalette.com, color-contrast-calc.
- **Specialist subprocess timeout as quorum failure** (research Rec 6). Node `spawn()` has no built-in timeout and trapped SIGTERM hangs indefinitely — `src/engine/claude/claude.exec.ts` already implements startup/stall/global timeouts with SIGTERM→SIGKILL escalation. Made the quorum semantics explicit: any rejection from `invokeClaude` (timeout, non-zero exit, spawn failure) counts identically as a "failed specialist." Added default 180 s per-call timeout (configurable, recommended 180–600 s), `trajectory.jsonl` entries for timeouts with `reason: "timeout"`, and a vitest that resolves one specialist and times out the other, asserting synthesis runs on the survivor. Sources: nodejs.org child_process docs, node issues #27639 / #51561, existing `claude.exec.ts` infrastructure.
- **Playwright peerDep version pin** (research Rec 7). `"playwright": ">=1.57.0 <2.0.0"` with `peerDependenciesMeta: { "playwright": { "optional": true } }`. 1.57 is where Chrome-for-Testing landed and `page.accessibility` was removed in favor of axe-core — this matches the rest of the spec's a11y approach. Sources: playwright.dev release notes, testdino.com, medium szaranger.
- **`engines.node: ">=20.0.0"`** (research Rec 8). Playwright deprecates Node 18 as of 1.54; Node 20 is LTS through mid-2026 and matches the `@types/node@22` dev-time baseline already present. Added as an acceptance criterion under the Version bump feature and listed in the CHANGELOG Breaking section. Flagged the mild conflict with constraints.md "engines field unchanged" — the field is currently absent, so adding it is introducing (not modifying) a constraint, and it is derived from the Playwright peer dep.
- **Future Considerations section** — captured items the refiner prompt says should not be added to the feature list: cross-invocation cache reuse via session continuity or persistent subprocess (research Rec 15), 1-hour cache TTL, a future `engines.node >=22` bump, headless-browser dashboard e2e, and additional sensor kinds (game/mobile/audio).

### Skipped

- None. All 16 Active Recommendations were incorporated — most directly as acceptance criteria, Rec 15 (session continuity) as a scope note within the prompt-caching feature plus a Future Considerations entry, and Rec 11 (drop `.ridgeline/cache-key.json`) is reflected both as an affirmative criterion ("no client-side cache-key file") and a removal from the original spec's invalidation criterion.

### Notes on fidelity to the refiner prompt

- `constraints.md` and `taste.md` were not modified, per the refiner prompt rule "Constraints are immutable."
- Research recommendations that would have required constraint changes (e.g. the Playwright peerDep version range, `engines.node`, the HSL brightening helper module) were expressed as acceptance criteria in `spec.md` feature sections rather than as edits to `constraints.md`. Where a research finding conflicted with an existing constraint (the `engines.node` bump vs. the "engines field unchanged" line in constraints.md), the conflict is flagged explicitly in the relevant spec feature.
- All additions cite the research source inline so subsequent refine iterations can see what came from research vs. original spec.
